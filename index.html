IOS Setup

How to update bundle identifier and app's name?


Setting up an iOS app requires the use of macOS, as it provides the necessary tools and software, such as Xcode, for building and testing iOS Apps.

Step 1 - Change app's bundle identifier

  1. Open the projects ios/ folder in Xcode
  2. In Xcode, navigate to the Runner project in the project navigator
  3. Select the Runner target
  4. Go to the General tab
  5. Update the Bundle Identifier to your new identifier (e.g., com.yourcompany.newapp)

  2. Find the key CFBundleNam and update its value to your new app name
    3.

  3. In the same file, you may also need to update the display name if it’s different from CFBundleName
    4. 
            <key>CFBundleDisplayName</key>
            <string>NewAppName</string>

Parse server is a Back-end as a service plataform, it was used as the back-end server of your app.

How to setup a parse server in Back4app?

Back4app was used to install the Parse server for your app.

Go to back4app.com signup and Create a project.

  3. Save it, we will use it later

Step 2 - Setup Live Query Url

  1. Access your parse server settings: App Settings > Sever Settings
  2. Click on Server URL and Live Query settings
  3. Enter your desired url and save
  4. Save it, we will use it later

Step 3 - Setup Cloud Code

  1. Unzip the file parse_server_config_files.zip Main files folder downloaded from codecanyon

How to setup IOS push notification?

Step 1 - Create an App ID

  1. Go to Apple Developer Account and log in with your Apple Developer credentials.
  2. Go to Certificates, Identifiers & Profiles:
    • In the left-hand menu, click on Certificates, Identifiers & Profiles.
  3. Create a New App ID:
    • Under Identifiers, click on the + button to create a new App ID.
    • Fill in the necessary details like the App Name and Bundle ID (e.g., com.example.myapp).
    • Ensure that Push Notifications is selected under the App Services section.
    • Click "Continue" and then Register to create the App ID.

Step 2 - Generate a Certificate Signing Request (CSR)

  1. Open Keychain Access:
    • On your Mac, open the Keychain Access application (you can find it via Spotlight Search or in the Utilities folder)
  2. Generate CSR:
    • In the Keychain Access menu, select "Keychain Access" > "Certificate Assistant" > "Request a Certificate from a Certificate Authority".
    • Enter your email address, and name, and select Saved to disk.
    • Click Continue and save the CSR file (e.g., CertificateSigningRequest.certSigningRequest).

Step 3 - Create the APNs Certificate

  1. Go to Certificates Section:
    • In the Apple Developer Account, go back to "Certificates, Identifiers & Profiles".
  2. Add a New Certificate:
    • Select "Apple Push Notification service SSL (Sandbox & Production)" and click "Continue".
    • Choose your App ID from the list and click "Continue".
  3. Upload CSR:
    • Upload the CSR file you generated earlier and click "Continue".
  4. Download Certificate:
    • Once the certificate is created, download the .cer file.

Step 4 - Install the APNs Certificate

  1. Open the .cer File:
    • Double-click the downloaded .cer file to install it in your Keychain Access.
  2. Export the Certificate:
    • In Keychain Access, find the installed certificate (it will have the name "Apple Development iOS Push Services" or similar).
    • Right-click the certificate and select "Export".
    • Save the certificate as a .p12 file and set a password when prompted.

Step 5 - Upload the APNs Certificate to firebase console

  1. Go to Firebase developer console.
  2. Select Your Project
  3. In the left-hand menu, click on the gear icon next to Project Overview and select Project settings.
  4. In the Project settings page, click on the Cloud Messaging tab.
  5. Scroll down to Apple app configuration.
  6. Find the APNs Certificates section
  7. Upload the .p12 for both development and production

How to setup Google social login plugin?

Step 1 - Add debug fingerprints to firebase

  1. Get fingerprints from your Android Studio debud keystore:
    • Run the Keytool Command
    • For macOS/Linux: keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android
    • For Windows: keytool -list -v -keystore %USERPROFILE%\.android\debug.keystore -alias androiddebugkey -storepass android -keypass android
    • For Release keystore: keytool -list -v -keystore [your release keystore path]
  2. After running the appropriate command for your operating system, locate the SHA-1, SHA-256 fingerprint in the output.
  3. Go to Firebase developer console.
  4. Select Your Project
  5. In the left-hand menu, click on the gear icon next to Project Overview and select Project settings.
  6. In General tab page scroll down to Your apps section
  7. Click on Add fingerprint, paste the SHA-1 and click on Save.
  8. Click again on Add fingerprint, paste the SHA-256 and click on Save.

Step 2 - Enable Google login in firebase console

  1. Go to Firebase developer console.
  2. Select Your Project
  3. In the left-hand menu, expand the Build section and click Authentication.
  4. In the Authentication page, click on the Get started button.
  5. Click on Google, Enable and then click Save
  6. Download the new google-services.json and GoogleService-info.plist

Step 3 - Setup Google login for Android

  1. Override the old file google-services.json by the new one in the following location: android/app/.

Step 4 - Setup Google login for IOS

  1. Override the old file GoogleService-info.plist by the new one in the following location: ios/Runner/.

Apple Pay

Step 1 - Certificate Signing Request (CSR) with the ECC algorithm and a key size of 256 bits

  1. Open your terminal
  2. Use the following command to generate an ECC private key with a key size of 256 bits:
    • openssl ecparam -genkey -name prime256v1 -out ec_private_key.pem
  3. Now, generate the CSR using the ECC private key:
    • openssl req -new -key ec_private_key.pem -out ec_csr.pem
  4. During this process, you’ll be prompted to enter information for the CSR, such as your country, state, organization, and so on. Fill in these details as required.

Step 2 - Setup a Merchant ID

  1. Login into Apple Developer account page..
  2. click on "Identifiers".
  3. Click on the "+" button in the top-right corner to add a new identifier..
  4. Select "Merchant IDs" and click "Continue".
  5. Enter a description for your merchant ID (e.g., "MyApp Merchant ID").
  6. Enter the Merchant ID in reverse domain name style (e.g., "com.mycompany.myapp.merchant").
  7. Click "Continue" to review the details.
  8. Click "Register" to complete the process.
  9. Configure the Merchant ID:
    • Go back to the "Identifiers" section and locate your newly created Merchant ID.
    • Filter the list by Merchant IDs
    • Click on your Merchant ID and click on Create certificate button then click on Continue
    • Upload the ec_csr.pem and click Continue

Step 3 - Enable Apple Pay in Your App

  1. Open your project in Xcode:
    • Go to the Signing & Capabilities tab.
    • Find Apple Pay capability.
    • Select the appropriate Merchant ID from the dropdown list.
  2. Open the file payment_configuration.dart located in the folder lib/configurations/, and updated the variable defaultApplePay content:
    • Set merchantIdentifier value to you Merchant ID.
  3. Open the files Runner.entitlements adn RunnerDebug.entitlements located in the folder: ios/Runner/ and the following code:
    4. 
          <key>com.apple.developer.in-app-payments</key>
          <array>

How to setup the Admin Panel?

Parse Server has Parse Dashboard But since it's not easy to use, we decided to offer an easy and modern Admin Panel for you.

To Install Dashboard is very easy and also it can be installed on any hosting provider, Shared, VPS, Dedicated, Cloud or others. Keep in mind that you need a Parse Server running somewhere, and at least 1 app is running, in order to get this Dashboard to work properly.

Step 1 - Upload Admin Panel Files to your hosting

  1. Extract admin_panel.zip you can found it in Main Files.zip
  2. Download the node_modules.zip and extract into the admin panel root folder
  3. Open Configs.php file and change: Website root url, AppId, RestApiKey and MasterKey of your Parse Server App.
  4. Upload in your hosting and Test by opening the url where you placed your admin panel in your browser.

Step 2 - Configure Admin Panel

Don't create user manualy in your parse dashboard, make sure you registered user from the app.

  1. Setup Admin User(s):
