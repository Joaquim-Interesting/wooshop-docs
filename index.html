Requirements

Install or Update flutter

Since this project is developed with Flutter, make sure to correctly install flutter and dart in your computer. If you need help to install flutter Click to Install Flutter


IDE or Text Editor

For all intallation process you will need an IDE oa a text editor, You can use Android Studio or Visual Studio Code, make sure to use latest versions.

Install Android Studio or Install Visual Studio Code

For this documentation we've used Android Studio.

Installation

Import and Setup Project

  1. Extract Zip file Downloaded from codecanyon
  2. Navigate to wooshop_app.zip file and extract it too
  3. Open android Studio to Import the project
  4. Click on Import Android Studio project
  5. Navigate to extracted wooshop_app folder
  6. Click in OK or import then wait for Android Studio to load and setup all files and index them.
  7. Open terminal or cmd
  8. In your project root run flutter pub get and wait to finish

Logo Settings

Customize Logo and Splash

  1. Go to assets>images and override icon.png and icon_dark.png with your own Logos
  2. In your project root from terminal run flutter pub run flutter_native_splash:create

Parse Server

Setup guide on configuring parse server installation for the app.

Parse server is a Back-end as a service plataform, it was used as the back-end server of your app.

How to setup a parse server in Back4app?

Back4app was used to install the Parse server for your app.

Go to back4app.com signup and Create a project.

Step 1 - Get a Back4app Keys

  1. Access your parse server settings: App Settings > Security & Keys
  2. Copy the Application ID, Client key and Master key
  3. Save it, we will use it later

Step 2 - Setup Live Query Url

  1. Access your parse server settings: App Settings > Sever Settings
  2. Click on Server URL and Live Query settings
  3. Enter your desired url and save
  4. Save it, we will use it later

Step 3 - Setup Cloud Code

  1. Unzip the file parse_server_config_files.zip Main files folder downloaded from codecanyon
  2. Access your parse server cloud code: Cloud Code > Functions & Web Hosting
  3. Delete the main.js file
  4. Upload the file package.json located in parse_server_config_files/Cloud code files/package.json
  5. Upload the file main.js located in parse_server_config_files/Cloud code files/package.json
  6. Click Deploy

How to connect parse server to the App?

Open the file app_config.dart located in the folder lib/configurations/, and fill the info listed bellow:

  • serverUrl - your parse server url (e.g: For back4app is https://parseapi.back4app.com/)
  • liveQueryUrl - your parse server livequery url
  • appId - your parse server App Id
  • clientKey - your parse server client key

How to create parse server config parameters?

  1. On your Parse server Dashboard go to More>config, and add the following parameters:
    2. Parameter name Description Data type Value
    agora_app_id Used to save the Agora.io App Id String Agora.io App Id
    appleLoginEnabled Used to enable or disable Apple Login Boolean true/false
    appSendGridEmail Used to save the sendgrid sender email String sendgrid sender email
    cashOnDeliveryEnabled Used to enable or disable Cash on delivery payment method Boolean true/false
    facebookLoginEnabled Used to enable or disable Facebook Login Boolean true/false
    googleAdsEnabled Used to enable or disable Google ads Boolean true/false
    googleAndApplePayGetWayEnabled Used to enable or disable Google and Apple pay payment getway Boolean true/false
    googleLoginEnabled Used to enable or disable Google login Boolean true/false
    paypalGatWayEnabled Used to enable or disable Paypal payment getway Boolean true/false
    paypalSandboxMode Used to enable or disable Paypal Sandbox mode Boolean true/false
    phoneLoginEnabled Used to enable or disable Phone number login Boolean true/false
    stripePaymentGetWayEnabled Used to enable or disable Stripe payment getway Boolean true/false
    paypalClientId Used to save the Paypal client id String Paypal client id
    paypalSecretKey Used to save the Paypal secret key String Paypal secret key
    sellerEarningsPercent Used to save the seller earning percentage String A number between 1 - 100
    sendGridApiKey Used to save the Sendgrid API key String Sendgrid API key
    stripePublishableKey Used to save the Stripe published key String Stripe published key
    stripeSecretKey Used to save the Stripe secret key String Stripe secret key

Push notification

Here is a setup guide on configuring push notification.

How to create and setup a Service account?

Step 1 - Create a Service Account

  1. Go to Firebase developer console.
  2. Select Your Project by clicking on the project you want to create a service account for
  3. In the left-hand menu, click on the gear icon next to Project Overview and select Project settings.
  4. In the Project settings page, click on the Cloud Messaging tab, and click on Manage Service Accounts link
  5. In next page click on the + CREATE SERVICE ACCOUNT button and follow the steps bellow:
    • Enter service account name and description, and click on the CREATE AND CONTINUE button
    • Select the role Owner and click on the CONTINUE button.
    • Click on the DONE button.

Step 2 - Add a key

  1. Click on the created service account's Action icon and select the option Manage keys.
  2. Click on the ADD KEY button and select the option Create new key
  3. Select JSON and click on the Create button.
  4. The JSON file will be downloaded automatically. Save this file in a secure location.

Step 3 - Add service account to the app

Open the file app_config.dart located in the folder lib/configurations/, and fill the info listed bellow:

  • serviceAccountJson - your service account json file's content
  • firebaseProjectId - your firebase project's id (you may find it in the service account json file)

  4. In the Project settings page, click on the Cloud Messaging tab.

  7. Copy the Key pair and save it

Open the file app_config.dart located in the folder lib/configurations/, and fill the info listed bellow:

  • pushGcm - your firebase project's Sender ID

How to setup IOS push notification?

Step 1 - Create an App ID

  1. Go to Apple Developer Account and log in with your Apple Developer credentials.
  2. Go to Certificates, Identifiers & Profiles:
    • In the left-hand menu, click on Certificates, Identifiers & Profiles.
  3. Create a New App ID:
    • Under Identifiers, click on the + button to create a new App ID.
    • Fill in the necessary details like the App Name and Bundle ID (e.g., com.example.myapp).
    • Ensure that Push Notifications is selected under the App Services section.
    • Click "Continue" and then Register to create the App ID.

Step 2 - Generate a Certificate Signing Request (CSR)

  1. Open Keychain Access:
    • On your Mac, open the Keychain Access application (you can find it via Spotlight Search or in the Utilities folder)
  2. Generate CSR:
    • In the Keychain Access menu, select "Keychain Access" > "Certificate Assistant" > "Request a Certificate from a Certificate Authority".
    • Enter your email address, and name, and select Saved to disk.
    • Click Continue and save the CSR file (e.g., CertificateSigningRequest.certSigningRequest).

Step 3 - Create the APNs Certificate

  1. Go to Certificates Section:
    • In the Apple Developer Account, go back to "Certificates, Identifiers & Profiles".
  2. Add a New Certificate:
    • Select "Apple Push Notification service SSL (Sandbox & Production)" and click "Continue".
    • Choose your App ID from the list and click "Continue".
  3. Upload CSR:
    • Upload the CSR file you generated earlier and click "Continue".
  4. Download Certificate:
    • Once the certificate is created, download the .cer file.

Step 4 - Install the APNs Certificate

  1. Open the .cer File:
    • Double-click the downloaded .cer file to install it in your Keychain Access.
  2. Export the Certificate:
    • In Keychain Access, find the installed certificate (it will have the name "Apple Development iOS Push Services" or similar).
    • Right-click the certificate and select "Export".
    • Save the certificate as a .p12 file and set a password when prompted.

Step 5 - Upload the APNs Certificate to firebase console

  1. Go to Firebase developer console.
  2. Select Your Project
  3. In the left-hand menu, click on the gear icon next to Project Overview and select Project settings.
  4. In the Project settings page, click on the Cloud Messaging tab.
  5. Scroll down to Apple app configuration.
  6. Find the APNs Certificates section
  7. Upload the .p12 for both development and production

Social Login Setup

WooShop comes with a lot of social login plugins, your users can register to your App with one click! Here is a setup guide on configuring social login.

How to setup Google social login plugin?

Step 1 - Add debug fingerprints to firebase

  1. Get fingerprints from your Android Studio debud keystore:
    • Run the Keytool Command
    • For macOS/Linux: keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android
    • For Windows: keytool -list -v -keystore %USERPROFILE%\.android\debug.keystore -alias androiddebugkey -storepass android -keypass android
  2. After running the appropriate command for your operating system, locate the SHA-1, SHA-256 fingerprint in the output.
  3. Go to Firebase developer console.
  4. Select Your Project
  5. In the left-hand menu, click on the gear icon next to Project Overview and select Project settings.
  6. In General tab page scroll down to Your apps section
  7. Click on Add fingerprint, paste the SHA-1 and click on Save.
  8. Click again on Add fingerprint, paste the SHA-256 and click on Save.

Step 2 - Enable Google login in firebase console

  1. Go to Firebase developer console.
  2. Select Your Project
  3. In the left-hand menu, expand the Build section and click Authentication.
  4. In the Authentication page, click on the Get started button.
  5. Click on Google, Enable and then click Save
  6. Download the new google-services.json and GoogleService-info.plist

Step 3 - Setup Google login for Android

  1. Override the old file google-services.json by the new one in the following location: android/app/.

Step 4 - Setup Google login for IOS

  1. Override the old file GoogleService-info.plist by the new one in the following location: ios/Runner/.
  2. Open the file GoogleService-info.plist and copy the content of the key REVERSED_CLIENT_ID
    3. 
          <key>REVERSED_CLIENT_ID</key>
          <string>Copy the content from here</string>
  3. Open the file info.plist located in: ios/Runner/.
  4. Paste the copied content on the first CFBundleURLSchemes value.
    5. 
          <key>CFBundleURLSchemes</key>
          <array>
              <string>Paste content right here</string>
          </array>

Theme Customization

We have include a Custom CSS File in the css Folder so that you can better handle your Customizations for New Styles or Overwriting Default Theme Styles. Simply add all your Custom CSS Codes in the css/custom.css File and link it in the Document <head> after the css/stylesheet.css Linking. Also make sure that this is the Last Linked CSS File in the Document <head> so that your Custom CSS Styles are Overwritten properly.

<head>
<!-- Stylesheet
============================== -->
<!-- Bootstrap -->
<link rel="stylesheet" type="text/css" href="assets/vendor/bootstrap/css/bootstrap.min.css" />
<!-- Font Awesome Icon -->
<link rel="stylesheet" type="text/css" href="assets/vendor/font-awesome/css/all.min.css" />
<!-- Magnific Popup -->
<link rel="stylesheet" type="text/css" href="assets/vendor/magnific-popup/magnific-popup.min.css" />
<!-- Highlight Syntax -->
<link rel="stylesheet" type="text/css" href="assets/vendor/highlight.js/styles/github.css" />
<!-- Custom Stylesheet -->
<link rel="stylesheet" type="text/css" href="assets/css/stylesheet.css" />

<!-- Here goes your custom.css
============================================= -->
<link rel="stylesheet" type="text/css" href="assets/css/custom.css" />
</head>

Inline code

Wrap inline snippets of code with <code>. Be sure to escape HTML angle brackets.

For example, <section> should be wrapped as inline.

For example, <code>&lt;section&gt;</code> should be wrapped as inline.

Code blocks

Use <pre>s for multiple lines of code. Once again, be sure to escape any angle brackets in the code for proper rendering. You may optionally add the .pre-scrollable class, which will set a max-height of 340px and provide a y-axis scrollbar.

For example:

<p>Sample text here...</p>
<p>And another line of sample text here...</p>

Have to use something like:

<pre><code>&lt;p&gt;Sample text here...&lt;/p&gt;
